Arduino Screaming Safe

The arduino screaming safe is, yes, safe that will only open if you scream the correct combination.

Arduino Screaming is a final project for SYDE 192. To open the safe you must "scream" at the safe at the correct times (in seconds) coresponding to the combination that you set. if the the screaming time is correct, the safe will open!

How Arduino Screaming Works

The screaming safe uses a number of hardware and software components. The main components are the, Arduino UNO microcontroller, the sound sensor and the laser cut box. A full parts list can be found in the README file.

The safe contains an arduino board that is programmed to detect a signal sent from the sound sensor whenever a sound above a specifc threshold is detected by the speaker. when a sound is detected a software interrupt void sound() is triggered. in the interrupt the code checks if the current time corresponds to the internal timer scaled to 30 seconds using the ISR(TIMER2_OVF_vect) interrupt. If the time matches, it will update the unlocked state. if all states are true then the safe will unlock by activating a servo motor. For more details about how the code works check out the README file.

Gangl, Design Case Study

in 2017, 90% of students reported that they were stressed and overworked during their post-secondary education. the Gangl App, is a platform designed and prototyped to combat stress caused by workload in undergraduate students.

The app was designed and prototyped using marvel design studio. the project followed the iterative design process to the book. it begain with the definition of a problem statement, identificaiton of key stakeholders and users and intensive reseach. A QFD and various other design tools were used to quantify specific aspects of the design. finally the project was designed from low fidelity, through high fidelity prototypes. It was presented as a final high fidelity prototype at the end of the term. Interested in learning more? View the full report.
